## 1. 项目概述
### 1.1 产品名称
**k6-config-traffic-simulator**（k6配置化业务流量模拟器）

### 1.2 产品定位
一款**零代码、配置驱动**的轻量级HTTP流量模拟工具，基于开源k6封装，无需编写测试脚本，仅通过YAML配置文件即可定义：多业务并行、自定义TPS负载曲线、多HTTP接口请求、灵活时长控制，完美模拟真实业务的HTTP访问规律。

### 1.3 项目背景
1. 原生k6需要编写JS脚本，非开发人员使用成本高；
2. 实际业务需要**多接口串联、多业务独立TPS、自定义负载曲线**并行模拟；
3. 需支持**短期压测**和**长期稳定性测试（无限运行）**两种场景；
4. 需求标准化：通过配置文件统一管理模拟规则，可复用、可版本控制。

### 1.4 核心目标
1. 零编码：用户仅需修改YAML配置，即可完成所有流量模拟规则定义；
2. 高灵活：支持多业务、多请求、多TPS曲线**并行独立运行**；
3. 强可控：支持自定义TPS曲线、有限时长/无限期运行；
4. 易使用：极简命令行操作，开箱即用，兼容k6原生监控与报告。

### 1.5 目标用户
1. 测试工程师：性能测试、稳定性测试、业务流量模拟；
2. 开发工程师：接口压测、服务容量评估；
3. 运维人员：生产环境预发布验证、长期流量值守；
4. 无k6编码基础的技术人员。

## 3. 详细功能需求
### 3.1 配置文件管理
1. 采用**YAML**作为配置文件格式，语法简洁、易读易写；
2. 配置文件支持：全局配置 + 多业务配置 + 单业务请求配置 + TPS曲线配置；
3. 支持配置文件校验：启动时自动校验格式、必填字段，报错提示清晰；
4. 支持通过命令行指定自定义配置文件路径。

### 3.2 全局运行配置
1. **运行模式**：
   - `limited`：有限时长，指定总运行时间（秒/分/时）；
   - `infinite`：无限期运行，直到手动终止（Ctrl+C）；
2. **全局参数**：日志级别、报告输出路径、是否开启详细请求日志；
3. **全局超时**：统一设置HTTP请求超时时间（可被业务级配置覆盖）。

### 3.3 多业务并行管理
1. 支持配置**多个独立业务**（如：登录业务、商品查询业务、下单业务）；
2. 每个业务拥有**独立的开关**（enable: true/false），可灵活启停；
3. 每个业务拥有**独立的TPS曲线**，互不干扰；
4. 所有启用的业务**并行执行**，模拟真实系统多业务同时访问的场景。

### 3.4 单业务多HTTP请求
1. 单个业务支持配置**多个HTTP请求**，默认**串行执行**（模拟用户操作流程）；
2. 支持标准HTTP方法：GET/POST/PUT/DELETE/HEAD；
3. 支持请求配置：URL、请求头、请求体（JSON/表单）、超时时间、断言；
4. 支持请求间参数传递（可选扩展：提取响应数据作为下一个请求参数）；
5. 支持思考时间：模拟用户操作间隔，更贴近真实业务。

### 3.5 自定义TPS曲线（核心功能）
1. 基于k6 `ramping-arrival-rate` 实现**精准TPS控制**；
2. 每个业务独立配置TPS曲线，支持**多阶段自定义**；
3. 支持的曲线类型：
   - 恒定TPS：固定请求速率持续运行；
   - 阶梯TPS：逐步升高/降低到目标TPS；
   - 浪涌TPS：峰值+低谷交替，模拟流量突刺；
   - 自定义曲线：任意多阶段组合（如：低负载→峰值→降载→恒定）；
4. 配置单位：`1s`（每秒请求数，即TPS）。

### 3.6 运行时长控制
1. **有限时长模式**：全局配置总运行时间，时间到后自动停止所有请求；
2. **无限期模式**：持续循环执行所有业务请求，直到用户手动终止；
3. 支持优雅退出：终止时等待当前请求完成，避免异常。

### 3.7 实时监控与报告
1. 控制台实时输出：总TPS、各业务TPS、平均响应时间、错误率、请求成功率；
2. 支持生成k6原生报告：HTML可视化报告、JSON数据报告；
3. 支持日志输出：请求详情、错误信息、运行状态。

### 3.8 命令行交互
极简命令行操作，核心命令：
```bash
# 默认加载 config.yaml 配置
k6 run simulator.js

# 指定自定义配置文件
k6 run simulator.js --config ./my-business.yaml

# 查看帮助
k6 run simulator.js --help
```

## 5. 配置文件规范（核心）
### 5.1 配置文件结构
```yaml
# -------------------------- 全局配置 --------------------------
global:
  run_mode: limited       # 运行模式：limited(有限时长) / infinite(无限期)
  total_duration: 300s    # 总时长（仅limited模式生效），支持s/m/h
  log_level: info         # 日志级别：debug/info/warn/error
  report_path: ./report   # 报告输出路径
  global_timeout: 10s     # 全局请求超时时间

# -------------------------- 业务列表（多业务并行） --------------------------
businesses:
  # 业务1：用户登录（独立TPS曲线）
  - name: user_login
    enable: true          # 启用业务
    timeout: 5s           # 业务级超时（覆盖全局）
    # TPS曲线配置（核心）
    tps_curve:
      time_unit: 1s       # 时间单位：1秒
      stages:             # 多阶段TPS曲线
        - target: 10      # 目标TPS
          duration: 60s   # 持续时间
        - target: 30
          duration: 120s
        - target: 20
          duration: 120s
    # 业务内多HTTP请求（串行）
    requests:
      - name: get_captcha
        method: GET
        url: http://localhost:8080/api/captcha
        headers:
          Content-Type: application/json
        timeout: 3s
      - name: post_login
        method: POST
        url: http://localhost:8080/api/login
        headers:
          Content-Type: application/json
        body: '{"username":"test","password":"123456"}'
        think_time: 1s    # 思考时间

  # 业务2：商品查询（独立TPS曲线，并行运行）
  - name: product_query
    enable: true
    tps_curve:
      time_unit: 1s
      stages:
        - target: 50
          duration: 300s
    requests:
      - name: list_product
        method: GET
        url: http://localhost:8080/api/product/list?page=1&size=10
```

### 5.2 关键字段说明
| 层级 | 字段 | 说明 | 必填 |
| ---- | ---- | ---- | ---- |
| global | run_mode | 运行模式：limited/infinite | 是 |
| global | total_duration | 有限模式总时长（如60s/5m） | limited模式必填 |
| businesses | name | 业务唯一名称 | 是 |
| businesses | enable | 是否启用该业务 | 是 |
| businesses.tps_curve | stages | TPS阶段列表（target=目标TPS，duration=持续时间） | 是 |
| businesses.requests | name | 请求名称 | 是 |
| businesses.requests | method | HTTP方法 | 是 |
| businesses.requests | url | 请求地址 | 是 |

## 6. 技术实现方案
### 6.1 整体架构
```
用户配置文件(YAML) → 配置解析器 → k6核心执行引擎 → 多业务并行模拟 → 监控&报告输出
```

### 6.2 核心技术实现
1. **配置解析**：使用k6内置的`k6/yaml`库解析YAML配置文件；
2. **动态生成k6脚本**：根据配置自动构建`scenarios`（多业务并行）、`options`（TPS曲线）；
3. **请求执行**：基于k6 `http`模块执行HTTP请求，支持所有HTTP特性；
4. **时长控制**：通过k6原生时长控制 + 无限循环实现两种运行模式；
5. **报告输出**：复用k6原生报告模块，无需二次开发。

### 6.3 项目目录结构
```
k6-config-traffic-simulator/
├── simulator.js        # 工具核心脚本（唯一执行文件）
├── config.yaml         # 默认配置文件
├── example/            # 示例配置（多业务、无限运行、浪涌曲线）
├── README.md           # 使用文档
└── report/              # 报告输出目录
```

## 7. 标准使用流程
1. **环境准备**：安装k6（官方一键安装脚本）；
2. **配置编写**：复制模板配置，修改业务接口、TPS曲线、运行模式；

6. **查看报告**：在报告目录获取HTML/JSON报告。

### 8.3 易用性验收
1. 非开发人员可在5分钟内完成配置并启动；

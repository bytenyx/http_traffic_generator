# HTTP流量发生器

一款轻量型HTTP请求生成工具，核心用于模拟实际环境基础流量，支持按自定义QPS曲线发送HTTP请求，可配置多场景并灵活切换，满足测试、验证目标服务在不同流量场景下的运行状态需求。

## 功能特性

### QPS曲线配置
- 支持固定QPS、阶梯曲线、线性曲线、自定义折线四种曲线类型
- 支持秒级QPS控制精度
- 支持设置任务总执行时长
- 支持QPS上下限设置

### 多场景管理
- 支持场景创建、保存、编辑、删除
- 支持场景切换
- 支持单个场景独立执行

### HTTP请求配置
- 支持GET、POST、PUT、DELETE四种请求方法
- 支持HTTP/HTTPS协议
- 支持配置请求头、请求体（JSON、Form格式）
- 支持设置请求超时时间

### 动态参数配置
- 支持时间戳、UUID、随机字符串等基础动态变量
- 支持导入CSV文件作为参数池
- 支持通过占位符插入动态变量

### 监控与统计
- 实时显示当前实际QPS、目标QPS
- 实时统计请求成功率、失败率
- 实时显示请求平均时延
- 任务结束后生成简易统计报表
- 支持导出Excel格式报表

### 执行引擎
- 支持单机运行模式
- 支持引擎启动、停止
- 支持设置CPU、内存使用上限
- 支持设置最大并发连接数

### 前置操作
- 支持为每个场景配置1-3个前置HTTP请求
- 支持从前置请求的响应结果中提取动态参数
- 支持JSON路径提取和正则表达式提取
- 支持提取的参数通过占位符插入主请求参数

## 技术栈

- **后端**: Node.js, Express
- **前端**: React, React Router
- **数据库**: 本地文件存储（JSON格式）
- **依赖库**:
  - 后端: express, body-parser, cors, uuid, papaparse, xlsx, jsonpath, systeminformation
  - 前端: react, react-dom, axios, react-router-dom, chart.js, react-chartjs-2

## 快速开始

### 安装依赖

```bash
# 安装后端依赖
npm install

# 安装前端依赖
cd client
npm install
```

### 构建前端项目

```bash
cd client
npm run build
```

### 启动服务器

```bash
# 在项目根目录运行
npm start
```

服务器将运行在 http://localhost:3001

## 使用方法

1. **访问应用**: 在浏览器中打开 http://localhost:3001
2. **创建场景**: 点击"新建场景"按钮，配置QPS曲线、HTTP请求、动态参数、前置操作等
3. **执行任务**: 在场景列表中选择场景，点击"执行"按钮启动引擎
4. **查看监控**: 点击"任务执行"按钮查看实时监控指标
5. **导出报表**: 在任务执行页面点击"导出Excel报表"按钮导出统计报表
6. **管理引擎**: 点击"引擎管理"按钮查看引擎运行状态和资源使用情况

## 注意事项

- 本工具仅支持单机运行模式，不支持分布式部署
- 建议在测试环境使用，避免在生产环境直接运行
- 当QPS较高时，请注意监控系统资源使用情况
- 前置请求最多支持3个，每个前置请求都可能影响任务启动时间

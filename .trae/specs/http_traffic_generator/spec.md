# HTTP流量发生器 - 产品需求文档

## 概述
- **Summary**: 一款轻量型HTTP请求生成工具，核心用于模拟实际环境基础流量，支持按自定义QPS曲线发送HTTP请求，可配置多场景并灵活切换，满足测试、验证目标服务在不同流量场景下的运行状态需求。
- **Purpose**: 解决测试人员和开发人员在接口压测、流量模拟验证方面的需求，提供简单易用的工具来模拟不同QPS曲线的HTTP请求。
- **Target Users**: 测试人员、开发人员（用于接口压测、流量模拟验证）。

## Goals
- 支持按设定QPS曲线（多种曲线类型）稳定发送HTTP请求，QPS控制精度满足测试需求。
- 支持多场景配置、保存与切换，可灵活配置HTTP请求参数及动态参数。
- 提供基础的监控统计能力，实时展示QPS、成功率、时延等指标。
- 操作界面简洁，配置流程清晰，无需复杂学习成本，支持快速创建、执行请求任务。

## Non-Goals (Out of Scope)
- 不实现用户真实行为特征（如思考时间、会话保持、业务链路串联、流量毛刺等）。
- 不实现网络环境模拟（延迟、丢包等）、客户端特征模拟（UA池、请求头差异化等）、流量回放能力。
- 不实现分布式执行引擎、容错与安全约束、扩展与集成相关功能。
- 仅支持本地Web页面接入，无需CLI命令行、Open API对接。

## Background & Context
- 本项目是一个轻量型HTTP请求生成工具，专注于基础流量模拟，无需复杂的真实环境仿真及工程化部署能力。
- 核心目标是支持按设定QPS曲线稳定发送HTTP请求，满足测试需求。

## Functional Requirements
- **FR-1**: QPS曲线配置模块 - 支持固定QPS、阶梯曲线、线性曲线、自定义折线四种曲线类型，支持秒级QPS控制精度，可设置任务总执行时长，支持立即执行、手动暂停/恢复/终止请求发送。
- **FR-2**: 多场景管理模块 - 支持场景创建、保存、编辑、删除，场景切换后可正常执行任务，支持单个场景独立执行。
- **FR-3**: HTTP请求配置模块 - 支持GET、POST、PUT、DELETE四种请求方法，支持HTTP/HTTPS协议，可配置请求头、请求体（JSON、Form格式）、请求超时时间。
- **FR-4**: 动态参数配置模块 - 支持时间戳、UUID、随机字符串等基础动态变量，支持导入CSV文件作为参数池，支持通过占位符插入动态变量。
- **FR-5**: 监控与统计模块 - 实时显示当前实际QPS、目标QPS、请求成功率、失败率、平均时延，任务结束后生成简易统计报表，支持手动导出Excel格式。
- **FR-6**: 执行引擎模块 - 支持单机运行模式，可手动启动、停止执行引擎，支持设置单机CPU、内存使用上限，支持设置最大并发连接数。
- **FR-7**: 前置操作与HTTP请求动态参数获取模块 - 支持为每个场景配置1-3个前置HTTP请求，支持从前置请求的响应结果中提取动态参数，支持JSON路径提取和正则表达式提取。

## Non-Functional Requirements
- **NFR-1**: 易用性 - 操作界面简洁，核心功能（创建场景、配置QPS、执行任务）流程不超过3步，支持配置项自动保存，提供简单的操作提示。
- **NFR-2**: 稳定性 - 在设定QPS范围内（≤1000 QPS），工具可稳定运行，无崩溃、卡死现象，请求发送过程中若出现网络中断，可自动停止发送，提示用户网络异常。
- **NFR-3**: 兼容性 - 支持在Windows 10及以上、macOS 12及以上系统运行，支持主流浏览器（Chrome 100+、Edge 100+、Firefox 95+）访问Web页面。

## Constraints
- **Technical**: 仅支持单机运行模式，无需分布式集群、K8s弹性扩缩能力，适配轻量型使用场景。
- **Business**: 简化设计，满足基础参数获取需求，前置请求限制为1-3个。
- **Dependencies**: 需安装基础Web服务依赖，提供一键启动脚本。

## Assumptions
- 用户仅需要基础的流量模拟能力，不需要复杂的真实环境仿真。
- 工具运行在本地环境，无需远程部署。
- 用户具备基本的HTTP请求知识，能够配置请求参数。

## Acceptance Criteria

### AC-1: QPS曲线配置验收
- **Given**: 用户配置了固定、阶梯、线性、自定义折线四种QPS曲线之一
- **When**: 执行任务时
- **Then**: 实际QPS与目标QPS偏差≤10%，时间粒度为秒级，无明显波动
- **Verification**: `programmatic`
- **Notes**: QPS上下限设置有效，超出限制时自动按上下限发送请求

### AC-2: 多场景管理验收
- **Given**: 用户创建、保存、编辑、删除场景
- **When**: 切换场景并执行任务
- **Then**: 场景切换后可正常执行任务，场景配置修改后保存生效
- **Verification**: `programmatic`

### AC-3: HTTP请求配置验收
- **Given**: 用户配置了四种请求方法之一，设置了请求地址、请求头、请求体、超时时间
- **When**: 执行任务时
- **Then**: 请求正常发送，请求超时设置有效，HTTPS接口可正常发送请求
- **Verification**: `programmatic`

### AC-4: 动态参数配置验收
- **Given**: 用户在请求参数中插入了动态变量（时间戳、UUID、随机字符串）或导入了CSV参数池
- **When**: 执行任务时
- **Then**: 动态变量自动替换为实际值，CSV参数按设置的取值方式调用
- **Verification**: `programmatic`

### AC-5: 监控与统计验收
- **Given**: 任务执行过程中
- **When**: 查看监控指标和任务结束后查看报表
- **Then**: 实时监控指标展示准确，任务结束后报表生成正常，数据准确，可成功导出Excel
- **Verification**: `programmatic`

### AC-6: 执行引擎验收
- **Given**: 用户通过Web页面按钮启动、停止引擎
- **When**: 引擎运行过程中
- **Then**: 引擎正常加载场景配置并发送请求，资源限制设置有效，日志可正常生成、保存
- **Verification**: `programmatic`

### AC-7: 前置操作与动态参数获取验收
- **Given**: 用户为场景配置了1-3个前置HTTP请求，并设置了参数提取规则
- **When**: 执行任务时
- **Then**: 前置请求按配置顺序执行，执行失败时主请求不启动，执行成功后可正常提取参数，提取的参数可通过占位符插入主请求参数
- **Verification**: `programmatic`

### AC-8: 非功能验收
- **Given**: 用户在指定系统、浏览器中操作工具
- **When**: 执行核心功能操作
- **Then**: 操作流程简洁，工具稳定运行，无崩溃、卡死现象，页面操作流畅，无兼容性异常
- **Verification**: `human-judgment`
